# YNX Web4 Public Testnet Demo Outline

## Goal

Deliver a three-minute sponsor demo that proves the system is live, protected, and already useful.

## Demo Structure

### 1. Opening: what YNX is

- show `https://www.ynxweb4.com`
- one sentence:
  - `YNX is a live public testnet for protected AI onchain execution and cross-chain test-asset routing.`

### 2. Live network proof

- open `https://www.ynxweb4.com/api/network/status`
- call out that RPC, EVM, REST, faucet, indexer, explorer, AI, and Web4 are online
- mention validator set restored to four bonded validators

### 3. AI protection proof

- show `/ai`
- explain Web4 session policy and protected `trade.execute`
- show AI preflight / settlement posture without overstating autonomy

### 4. Bridge readiness proof

- show `/readiness`
- state:
  - `4/5 deposit-tested`
  - `5/5 some form of release proof observed`
  - `2/5 automatic-loop-ready`
  - `2/5 strongest observed automatic public evidence today`
- explain BSC as the clearest remaining automation blocker, and frame BTC/TRON as configured but still lighter on public observed automation evidence than Sepolia

### 5. Closing ask

- ask for grant or sponsorship, not token financing
- use a narrow ask:
  - infra and adapter completion
  - audit and reliability budget
  - builder onboarding and ecosystem demo support

## Suggested Closing Line

`We are not pitching a concept. The public testnet is already live, protected, and measurable. We are asking for ecosystem funding to complete the remaining route automation and production hardening.`
