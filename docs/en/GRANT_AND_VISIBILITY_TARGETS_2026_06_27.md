# YNX Grant and Visibility Targets

Status: active working shortlist  
Prepared on: 2026-06-27  
Source policy: official public sources only

## 1. Goal

This document turns YNX's current truthful state into a practical shortlist for:

- grant applications
- ecosystem support requests
- builder-program visibility
- outreach prioritization

It is not a promise that every listed program is a fit today. The point is to
separate:

- active official programs worth pursuing now
- programs worth pursuing later after additional integration work
- programs that are currently poor fits

## 2. Current YNX posture that matters for grants

YNX currently has the strongest story in:

- public-testnet infrastructure
- AI/Web4 policy-bounded execution
- settlement and machine-payment rails
- traceability / accountability / forensics
- public-goods style tooling and operator surfaces

YNX is not yet strongest as:

- a production custody or redemption network
- a completed multi-route bridge success story
- a finished multi-chain app consumer product

That means YNX should target programs that value:

- open infrastructure
- builder tooling
- public goods
- AI / agent / execution tooling
- research, security, and protocol-adjacent developer surfaces

## 3. Tier 1 targets — best fit now

### 3.1 Ethereum Foundation ESP

Official sources:

- <https://esp.ethereum.foundation/>
- <https://esp.ethereum.foundation/about>
- <https://esp.ethereum.foundation/applicants>

Why it fits:

- ESP explicitly supports public goods, infrastructure, research, and builder
  tooling
- YNX now has a clearer public-goods angle in:
  - accountability / forensics infrastructure
  - policy-bounded agent execution
  - developer/operator infrastructure

Best framing for YNX:

- open-source accountability / forensics surfaces
- AI/Web4 constrained execution and settlement infrastructure
- operator tooling and safer machine-action patterns

Needed discipline:

- do not pitch YNX as a finished general-purpose multi-chain product
- emphasize infrastructure and public-goods contribution
- keep the bridge story testnet-scoped and evidence-based

### 3.2 Base funding pathways

Official sources:

- <https://docs.base.org/get-started/get-funded>
- <https://base.org/>

Why it can fit:

- Base documents multiple builder funding pathways
- YNX has a possible angle around:
  - agent payments
  - developer-facing execution rails
  - accountability tooling

Best framing for YNX:

- wallet / agent / payment developer tooling
- safer AI action boundaries
- operator-facing trace and evidence infrastructure

Caution:

- this is a better fit if YNX presents concrete Base-facing integration or
  tooling value, not just a generic chain pitch

### 3.3 Optimism / Superchain contributor funding

Official sources:

- <https://atlas.optimism.io/>
- <https://gov.optimism.io/c/grants/87>

Why it can fit:

- Optimism keeps active public-goods and grants/funding surfaces
- YNX can align if it frames work as open infrastructure or contributor tooling

Best framing for YNX:

- public-goods accountability tooling
- safer autonomous execution patterns
- reusable open tooling for onchain operators/builders

Caution:

- stronger fit if YNX contributes something clearly useful to the wider
  Superchain / OP Stack world, not only to YNX itself

## 4. Tier 2 targets — fit later after more integration

### 4.1 Solana Foundation funding

Official source:

- <https://solana.org/grants-funding>

Why it is interesting:

- the program supports public goods and milestone-based grants
- YNX's accountability / tracing direction could eventually translate into
  Solana-facing tooling or research

Why it is not Tier 1 today:

- YNX's strongest current live proof is not Solana-native
- a stronger application would likely need explicit Solana integration,
  adapters, or tooling value

### 4.2 Aptos Foundation grants

Official sources:

- <https://aptosfoundation.org/grants>
- <https://aptosfoundation.org/grants/ecosystem>

Why it is interesting:

- Aptos grants explicitly mention tools, payment infrastructure, and user
  experiences
- YNX has relevant machine-payment / execution / accountability building blocks

Why it is not Tier 1 today:

- YNX does not yet present Aptos as a primary live integration story
- better fit after concrete Aptos-facing tooling or execution support exists

### 4.3 Avalanche grants

Official source:

- <https://build.avax.network/grants>

Why it is interesting:

- Avalanche publicly lists grants and funding routes
- open-source, research, or infrastructure angles could fit later

Why it is not Tier 1 today:

- YNX would need a clearer Avalanche-specific integration or reusable infra
  contribution

## 5. Poor fit or not a priority right now

### 5.1 Arbitrum Foundation grant program

Official source:

- <https://arbitrum.foundation/grants>

Current official signal:

- the Arbitrum Foundation Grant Program page currently marks the program
  "Inactive"

Conclusion:

- do not treat this as an active primary target right now

### 5.2 Broad "we support every chain" outreach

Why not:

- YNX's strongest current story is disciplined and specific
- broad chain-agnostic pitching weakens truthfulness
- current bridge reality is still `2/5 full_loop_tested`

Conclusion:

- do not oversell universality
- apply only where YNX can explain exact ecosystem value
