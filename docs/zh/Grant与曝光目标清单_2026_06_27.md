# YNX Grant 与曝光目标清单

状态：active working shortlist  
整理日期：2026-06-27  
来源要求：仅使用官方公开来源

## 1. 目标

这份文档把 YNX 当前最真实的状态，转成实际可执行的：

- grant 申请目标
- ecosystem support 申请目标
- builder program 曝光目标
- 对外优先级排序

它不是说“所有列出来的都适合现在就投”，而是把目标拆成：

- 现在就值得优先推进的
- 再做一些集成后更适合推进的
- 当前不适合或不优先的

## 2. 当前 YNX 最适合拿去申请的定位

YNX 当前最强的点在：

- public-testnet infrastructure
- AI / Web4 policy-bounded execution
- settlement / machine payment rails
- traceability / accountability / forensics
- public-goods 风格的 tooling 与 operator surface

YNX 当前不最强的点在：

- 生产级 custody / redemption network
- 已经完全完成的多路线 bridge success story
- 已经收官的多链 consumer app 产品

所以当前最适合的 grant 类型应该偏向：

- open infrastructure
- builder tooling
- public goods
- AI / agent / execution tooling
- research / security / protocol-adjacent developer surfaces

## 3. Tier 1 —— 现在最值得优先推进

### 3.1 Ethereum Foundation ESP

官方来源：

- <https://esp.ethereum.foundation/>
- <https://esp.ethereum.foundation/about>
- <https://esp.ethereum.foundation/applicants>

为什么适合：

- ESP 明确支持 public goods、infrastructure、research、builder tooling
- YNX 现在已经比较适合从下面几个方向讲：
  - accountability / forensics infrastructure
  - policy-bounded agent execution
  - developer / operator infrastructure

YNX 最好的申请表述：

- open-source accountability / forensics surface
- AI / Web4 constrained execution and settlement infrastructure
- safer machine-action pattern 和 operator tooling

需要注意：

- 不要把 YNX pitch 成已经完成的通用 multi-chain 产品
- 应该强调 infrastructure / public goods 贡献
- bridge 叙事继续严格限制在 testnet evidence 范围

### 3.2 Base funding pathways

官方来源：

- <https://docs.base.org/get-started/get-funded>
- <https://base.org/>

为什么可能适合：

- Base 明确给了多种 builder funding path
- YNX 可以从下面的角度切：
  - agent payments
  - developer-facing execution rails
  - accountability tooling

更好的申请表述：

- wallet / agent / payment developer tooling
- safer AI action boundary
- operator-facing trace / evidence infrastructure

注意：

- 只有在 YNX 能明确说清楚自己对 Base builder 的具体价值时，这个目标才强
- 不能只是泛泛地说“我们也是一条链”

### 3.3 Optimism / Superchain contributor funding

官方来源：

- <https://atlas.optimism.io/>
- <https://gov.optimism.io/c/grants/87>

为什么可能适合：

- Optimism 仍然有公开的 funding / grants 入口
- 如果把 YNX 讲成 open infrastructure / contributor tooling，会有一定契合度

更好的申请表述：

- public-goods accountability tooling
- safer autonomous execution pattern
- reusable open tooling for onchain operator / builder

注意：

- 如果只是对 YNX 自己有用，而不是对更广的 Superchain / OP Stack 世界有用，
  适配度就会下降

## 4. Tier 2 —— 以后更适合

### 4.1 Solana Foundation funding

官方来源：

- <https://solana.org/grants-funding>

为什么值得关注：

- 这是公开且活跃的 funding program
- 支持 public goods 和 milestone-based grants

为什么现在还不是 Tier 1：

- YNX 当前最强 live proof 不是 Solana-native
- 要更强地去投，最好先有更明确的 Solana adapter / tooling / integration

### 4.2 Aptos Foundation grants

官方来源：

- <https://aptosfoundation.org/grants>
- <https://aptosfoundation.org/grants/ecosystem>

为什么值得关注：

- Aptos grants 明确提到 tools、payment infrastructure、user experience
- YNX 的 machine payment / execution / accountability 方向有一定相似性

为什么现在还不是 Tier 1：

- YNX 还没有把 Aptos 作为 primary live integration story
- 更适合在有 Aptos-facing tooling 之后再重点推进

### 4.3 Avalanche grants

官方来源：

- <https://build.avax.network/grants>

为什么值得关注：

- Avalanche 有公开 grants / funding 入口
- 开源、research、infra 方向后续可能有契合点

为什么现在还不是 Tier 1：

- 还缺一个足够明确的 Avalanche-specific integration 或 reusable infra 价值

## 5. 当前不适合或不优先

### 5.1 Arbitrum Foundation grant program

官方来源：

- <https://arbitrum.foundation/grants>

当前官方信号：

- 页面直接标了 "Inactive"

结论：

- 当前不要把它当作 active primary target

### 5.2 宽泛的“我们支持所有链”式曝光

为什么不该这么做：

- YNX 当前最强的故事是具体而克制的
- 泛化到“全都支持”会削弱可信度
- 当前 bridge 最强事实仍然只是 `2/5 full_loop_tested`

结论：

- 不要过度宣称 universality
- 只在能说明 exact ecosystem value 的地方去投
