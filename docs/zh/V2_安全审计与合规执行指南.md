# YNX v2 安全审计与合规执行指南

状态：active  
赛道：`v2-web4`  
最后更新：2026-03-21

## 1. 目标

把 YNX 准备到“你只需要去平台填表提交”的状态。

本阶段要产出的结果：

- 审计范围与架构证据完整
- 合规准备材料完整（SOC 2 / ISO 27001 前置）
- 证据包可重复生成
- 一条命令可重新打包提交材料

## 2. 并行推进原则

两条线同时跑：

- 安全审计线（主线）
- 合规准备线（并行）

不要串行等待，按周同步验收。

## 3. 前 4 周执行

## 第 1 周：范围冻结 + 架构定稿

必做：

1. 用 git tag 冻结提交范围。
2. 锁定核心文档和 API：
   - `docs/en/YNX_v2_WEB4_SPEC.md`
   - `docs/en/YNX_v2_AI_SETTLEMENT_API.md`
   - `docs/en/YNX_v2_WEB4_API.md`
   - `infra/openapi/ynx-v2-ai.yaml`
   - `infra/openapi/ynx-v2-web4.yaml`
3. 锁定加入手册（EN/ZH）。
4. 跑文档验收脚本：
   - `scripts/verify_docs_readiness.sh`

退出条件：

- tag 已创建
- 文档验收 `13/13 PASS`

## 第 2 周：威胁建模 + 风险台账 + 控制映射

必做：

1. 威胁建模覆盖：
   - 密钥泄露
   - 验证人双签 / 掉线
   - faucet 滥用
   - `/web4/*` 权限提升
   - `/ai/*` 结算争议滥用
2. 风险台账（严重级别、责任人、修复截止时间）。
3. 合规控制初版（权限、变更、日志、应急、密钥）。

## 第 3 周：预审加固 + 证据采集

必做：

1. 复现部署与验收流程：
   - `chain/scripts/v2_public_testnet_deploy.sh`
   - `chain/scripts/v2_public_testnet_verify.sh`
   - `chain/scripts/v2_public_testnet_smoke.sh`
2. 采集对外服务健康证据（RPC/EVM/Faucet/Indexer/Explorer/AI/Web4）。
3. 采集验证人和 watchdog 运行证据。
4. 生成提交包：
   - `scripts/prepare_audit_compliance_pack.sh`

## 第 4 周：提交流程闸门

必做：

1. 审计提交包完成复核。
2. 合规提交包完成复核。
3. 最终提交 tag 创建并推送。

## 4. 平台建议

安全审计优先批次：

- OpenZeppelin
- Trail of Bits
- Halborn

并行市场化补充：

- Cantina
- Code4rena
- Immunefi（审计后长期漏洞赏金）

合规平台：

- 自动化：Drata / Vanta / Secureframe（三选一）
- 审计机构：Schellman / A-LIGN（二选一）

## 5. 出发前最终闸门（Go / No-Go）

全部满足才允许外发：

- 文档验收 `13/13 PASS`
- 最新 verify 输出已归档
- 最新 smoke 输出已归档
- release 说明与当前变更一致
- 提交 tag 已推远端且不可变更

任一项失败即 No-Go，先修复再提审。
