状态：active

## 1. 目标

让你可以直接拿仓库内容去投审计/合规平台，尽量减少临场编写。

## 2. 一条命令生成提交包

在仓库根目录执行：

```bash
./scripts/prepare_audit_compliance_pack.sh
```

会自动产出：

- 文档完备性验收报告
- 公网运行证据报告
- 审计与合规提交包模板
- `output/` 下的压缩包

## 5. 各平台提交流程清单

## 安全审计公司（OpenZeppelin / Trail of Bits / Halborn）

- [ ] 填仓库地址
- [ ] 贴提交包 C 章节审计范围
- [ ] 贴提交包 D 章节风险重点
- [ ] 附文档验收和运行证据
- [ ] 填审计时间窗口和预算模式

## 审计竞赛（Cantina / Code4rena）

- [ ] 提交不可变 tag + commit hash
- [ ] 明确 in-scope / out-of-scope 路径
- [ ] 给奖励策略和严重级别规则
- [ ] 给参赛者快速复现实验命令

## 漏洞赏金（Immunefi）

- [ ] 公布范围与排除项
- [ ] 公布严重级别与奖励表
- [ ] 公布披露流程和响应 SLA
- [ ] 链接 `SECURITY.md` 与应急联系人

## 合规自动化（Drata / Vanta / Secureframe）

- [ ] 建控制项责任人
- [ ] 上传变更管理证据（tag/release）
- [ ] 上传日志与访问控制证据
- [ ] 上传应急和备份流程证据

## 合规审计机构（Schellman / A-LIGN）

- [ ] 提交架构、范围和准备包
- [ ] 明确首期目标（SOC2 Type I / ISO27001 准备）
- [ ] 明确证据采样周期要求

## 6. 最终提交闸门

全部满足再提交：

- 文档验收 PASS
- 运行证据 PASS
- 提交 tag 已推远端
- 提交包联系人/商务字段已填完
