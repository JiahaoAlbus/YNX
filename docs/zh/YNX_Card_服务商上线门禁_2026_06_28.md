# YNX Card 服务商上线门禁

状态：active pre-go-live gate  
整理日期：2026-06-28  
用途：定义 YNX Card 从 mock / provider-readiness 走到真实 sandbox 或 production provider 路径前必须满足的最低条件

## 1. 当前位置

YNX Card 现在是 programmable mock/control layer。

这已经足够证明：

- 基于规则的授权判断
- bounded AI-agent spend control
- 明确的 approve / decline 结果
- 可审计、可复核的事件记录

但这还不足以宣称：

- 真实 issuer-backed card
- 真实 PAN/CVV 处理
- live processor/network authorization
- production settlement / dispute operation

## 2. 门禁阶段模型

YNX Card 应该按下面顺序前进：

1. mock logic proven
2. provider sandbox technically mapped
3. legal/compliance ownership assigned
4. security and operational controls proven
5. limited provider pilot
6. broader production rollout

不要从阶段 1 直接跳到阶段 5。

## 3. 进入 provider sandbox 前必须满足的门禁

在真实 sandbox integration 之前，YNX 至少需要：

- 法律实体和签字权
- 明确的 provider contracting owner
- 明确的 KYC/KYB/AML 责任划分 owner
- 明确的 incident response / customer support owner
- provider API contract review
- provider event/reference-id mapping design
- provider credential 的安全存储方案
- 明确的 mock / sandbox / production 环境隔离

## 4. 进入 limited real pilot 前必须满足的门禁

在有限真实 pilot 前，还需要：

- YNX 规则与 provider rail 的授权映射文档
- production-grade audit log retention 方案
- reconciliation ledger 和 exception handling workflow
- refund / reversal / chargeback 流程
- sanctions / fraud / dispute escalation path
- 明确 PCI scope decision
- 隐私与数据处理边界审查
- freeze / resume / incident handling operator runbook
- settlement mismatch 的 finance/treasury ownership

## 5. 进入更广 production claim 前必须满足的门禁

在更广泛 production claim 前，还需要：

- 外部安全审查或 provider 要求的同等级审查
- 超过 demo 级别 json/local 假设的 durable persistence
- rollback 和 incident-response drill
- provider SLA / support model 确认
- 全站文案和文档法律口径更新
- 对外口径不再依赖“mock-only”解释

## 6. 硬性 no-go 条件

如果下面任意一点还为真，就绝不能把 YNX Card 说成 live-go-ready：

- 没有法律实体
- 没有签好的 provider relationship
- 没有明确的 KYC/KYB/AML ownership
- 没有 PCI/data-scope decision
- 没有 reconciliation/dispute ownership
- 没有 provider event-id mapping
- 没有 secure credential storage
- 没有 incident-response owner

## 7. 当前 YNX 在这些门禁上的真实状态

当前诚实状态：

- phase 1：yes，mock logic proven
- phase 2：部分具备，已经有 provider-readiness packet 和 API/control mapping
- phase 3：未完成
- phase 4：未完成
- phase 5：未开始
- phase 6：未开始

所以当前正确对外口径是：

- YNX Card 已经 provider-ready for review
- YNX Card 还不是 provider go-live ready
