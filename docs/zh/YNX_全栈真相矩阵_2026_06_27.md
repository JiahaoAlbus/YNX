# YNX 全栈真相矩阵

状态：active  
编写时间：2026-06-27  
用途：用一页讲清楚哪些现在就能跑、哪些只是 mock、哪些是 public testnet、哪些还依赖 provider / 法务 / 主网级工作

## 1. 为什么需要这份文档

YNX 现在已经同时覆盖：

- chain 和 validator 基础设施
- public bridge route
- Web4 policy/session 授权
- AI settlement 与 agent execution
- protected trace / accountability 流程
- YNX Card mock 控制层
- grant / outreach / compliance 材料

这些都是真的，但它们不在同一个成熟度层级。

这份文档就是最短、最诚实的回答：

- 哪些已经线上可见
- 哪些代码里已经有，但线上部署还没完全跟上
- 哪些是刻意的 mock
- 哪些还需要外部 provider、法务、审计或主网级建设

## 3. 按成熟度快速看

### 3.1 已经 live 的 public testnet 能力

- chain / RPC
- indexer
- explorer
- website
- Web4 hub
- AI gateway
- public bridge readiness board

### 3.2 仓库里已经有，但线上部署颗粒度更窄

- AI `/health` 和 `/ready` 里更新的 forensics counter
- protected trace / case detail workflow
- 比 public explorer 更细的 provenance-anchor 与 case-review 细节

### 3.3 可运行的 mock / proof layer

- YNX Card Mock
- 通过 mock/provider 未补齐 rail 承载的 bounded AI-agent spending

### 3.4 没有外部对手方就还不完整的部分

- issuer-backed 真 card program
- 生产级银行 / processor / network 集成
- 主网级法务和审计包
- 所有 bridge route 的 full completion

## 4. 关于“像人民币一样编号”的正式结论

YNX 现在应该继续坚持这个架构决议：

- 用 `lot lineage + pro-rata taint tracking` 作为 V2 真相底座
- 在真实存在来源锚点的地方增加稳定 provenance anchor
- 在它上面叠加 case、risk、clustering、graph、evidence 层
- 不要把系统描述成 merge / split 之后每个最小币碎片还保留永久单体 serial tracking

也就是说，更大的 accountability / forensics-engine 策略已经是 YNX
正确的官方方向，但它应该作为 V2 的继续补强，而不是另起一个脱离现有系统的重写。

## 5. 现在最安全的对外说法

可以说：

- YNX 是 live public-testnet 的 Web4 + AI execution stack
- YNX 有真实 bridge route evidence，但不同路线成熟度不一样
- YNX 有针对 traceable asset 的 protected accountability / forensics 能力
- YNX Card 当前证明的是 programmable control logic 的 mock layer，不是 live issuer-backed payment card

不能说：

- 所有 bridge routes 都 complete / automatic
- 每个 token 单位都有永久终身 serial number
- YNX 已经在跑持牌 production card program
- YNX 已 fully audited、legally complete、institution-ready
