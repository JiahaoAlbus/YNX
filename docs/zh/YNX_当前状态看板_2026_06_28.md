# YNX 当前状态看板

状态：active  
整理日期：2026-06-28  
用途：用最短的话回答哪些已经 live、哪些只是 mock、哪些是本地能力强于线上、哪些仍依赖外部条件

## 1. 一屏总表

| 模块 | 当前状态 | 最短真相 |
|---|---|---|
| Chain / RPC / REST / EVM | Live public testnet | 现在已经在线可用 |
| Indexer / Explorer | Live public testnet | 公共链数据与 redacted trace preview 已在线 |
| Web4 policy/session | Live + local | 受保护的 policy/session gating 已部署且可跑 |
| AI settlement | Live + local | vault/job/finalize 流程已在 public testnet 上可跑 |
| Bridge | 混合成熟度 | 架构真实，但今天只有 `2/5` route 是 `full_loop_tested` |
| Trace / accountability | 已实现，部分受保护 | 全能力存在，但公开侧是刻意做了 redaction |
| YNX Card Mock | Runnable mock | 逻辑证明是真实的，但 issuer-backed card 还没 live |
| AI Agent Spending | Repo/live hybrid | 有 bounded spending 逻辑，但真实 provider spend rail 还没 live |
| Grant / 对外材料 | 现在就可用 | 已经适合给 grant、sponsor、provider、技术尽调用 |
| Compliance / legal | 边界已写清 | 诚实边界已具备，但法律实体和正式 sign-off 还是外部工作 |

## 2. 现在已经 live 的东西

- public-testnet chain 和 RPC
- EVM RPC 和 REST
- indexer 和 explorer
- Web4 Hub
- AI Gateway
- 官网
- bridge readiness board
- 至少 `2/5` bridge route 有 `full_loop_tested` 证据

## 3. 现在能跑，但对外仍要谨慎表述的东西

- protected trace graph 和 forensics cases
- comparative taint model 与 lot lineage
- `issuance_id` / `deposit_batch_id` 这类 provenance anchor
- YNX Card Mock 的授权逻辑
- AI-agent bounded spending model

这些都是真实 repo 能力，但有些是受保护接口，有些只通过 mock/testnet flow 表达，不等于公开匿名可用。

## 4. 今天仍然属于 mock 的东西

- YNX Card 作为真实 issuer-backed card product
- 接到真实卡网络上的 programmable authorization
- 真正的 KYC / issuer / processor / settlement integration

安全表述：

- YNX Card 目前证明的是 programmable control logic 和 auditability
- 不是已经上线的受监管真实发卡项目

## 5. 仍然被外部条件卡住的东西

- bridge `5/5 full_loop_tested`
  - 当前 blocker 是明确的 signer / lockbox / route config 问题
- 真实 issuer / processor / network-backed YNX Card
- 正式法律实体和公司级 legal sign-off
- 外部安全审计发布
- mainnet-grade 的可靠性、持久化、运维硬化

## 6. 今天不能说的话

不要说：

- 所有 bridge route 都已经完成
- 所有 bridge route 都自动可释放
- YNX Card 已经是 live bank-card program
- YNX 已经法律完成或 fully audited
- YNX 已经是 mainnet-grade production financial network
