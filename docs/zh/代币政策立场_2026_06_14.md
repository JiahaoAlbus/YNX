# YNX 代币政策立场

状态：active financing policy note  
最后更新：2026-06-14  
规范语言：中文

## 目的

这份文件把当前融资口径，与仓库里较早期的 v0 tokenomics 草案和测试网经济设
计说明分开，避免投资人把历史草案直接当成当前正式条款。

## 投资人今天应该怎么理解

- YNX 不是靠“代币涨价故事”融资。
- NYXT 应被表述为 utility / coordination asset，而不是投资回报承诺。
- 仓库中的历史 tokenomics 草案，不等于最终主网或当前融资 term sheet。

## 当前更安全的融资口径

在正式治理、法务和披露流程完成前：

- 不应把 founder fee 当成默认生产政策；
- 未来默认 fee policy 应保持 `fee_founder_bps = 0`，除非另有单独论证和治理程序；
- team / treasury / community allocation 仍应被视为未最终定稿的设计空间，除非在经批准的融资文件里被重新明确；
- 对外融资应强调基础设施收入，而不是 token rent。

重要的线上现状补充：

- 截至 2026-06-14，公开 indexer overview 仍暴露测试网运行参数
  `fee_founder_bps = 1000`；
- 这个值应被如实表述为当前测试网 runtime state，而不是投资人今天被要求
  按之承保的正式融资政策；
- 如果线上参数和融资口径不同，YNX 应明确解释差异来源，而不是让投资人自行猜测。
