# YNX 基础设施角色图

状态日期：2026-06-13

## 当前线上机器分工

### `43.153.202.237` 主服务器

- 生产 validator
- 公网 RPC / EVM / faucet / indexer / explorer
- AI gateway
- Web4 hub
- bridge service
- Caddy ingress

这台机器现在是主执行面，最适合继续专注生产流量和核心公网服务。

### `43.134.23.58` 新加坡

- bonded validator
- 独立 ops observer
- 带快照的恢复节点

这台机器现在会定时写出独立观测快照：

- `/var/lib/ynx-ops-observer/latest.json`

同时它也开始保留链快照，后面恢复 validator 会更快，不需要一切都依赖主节点。

### `43.162.100.54` 硅谷

- bonded validator
- 保持为轻量共识节点

这台机器之前磁盘压力最大，已经清出安全余量。后续最适合继续保持“纯 validator”角色。

### `43.164.132.81` 首尔

- bonded validator
- 读流量副本候选
- 带快照的 peer

这台机器对外有读端口能力，适合承担 backup read surface / recovery source。

## 当前最合理的用法

- 主服务器：执行面和公网产品面
- 新加坡：观测、快照、恢复支持、validator
- 硅谷：稳定 validator
- 首尔：validator + 读副本 + 快照
