## 3. 500 词长版

`YNX 是一个早期开源基础设施项目，重点是 AI-native execution、
 policy-bounded onchain actions 和 developer-facing Web4 tooling。这个
 项目当前最强的地方不是 token story，而是公开技术资产。YNX 已经有公
 开官网、公开仓库、公开技术与尽调材料，以及 RPC、indexer、explorer、
 AI gateway、Web4 等真实公网测试网证据。

YNX 解决的问题很直接：很多早期 crypto 和 agentic 基础设施难以核验、
 难以集成，而且对真实就绪度的表述往往比事实更激进。开发者需要的是更
 容易验证、更容易使用、并且在执行边界上更克制的基础设施。YNX 试图通
 过 policy controls、session authorization、AI settlement rails 和
 builder-facing infrastructure 去提供这一层。

诚实边界同样重要。YNX 不应被表述为 production-grade 主网金融基础设
 施、托管平台或已经完成公司化的主体。当前没有公开法律实体，没有公开
 外部审计报告，也没有公开法律意见书。因此，YNX 当前最适合的是 grants、
 赞助、战略基础设施支持，以及理解公司化工作尚未完成的探索性早期投资
 沟通。

YNX 现在值得支持的原因在于：它已经不是概念阶段，技术可行性已经在公
 开环境中被证明。下一步不是证明概念，而是把早期基础设施打磨成更强的
 产品包装、运维成熟度、生态匹配度和公司化基础。资金最该用于安全审查
 与修复、监控和灾备、builder tooling 和文档、外部 operator 引入，以
 及法律和政策基础设施。`

## 4. 资金用途统一口径

- 安全审查、修复和 incident-response hardening；
- 监控、告警、备份和灾备；
- builder tooling、SDK、文档和集成流程；
- validator diversity 和外部 operator onboarding；
- 法律结构、terms、privacy 和合规落地；
- 少量试点客户或生态合作接入。

## 5. 当前边界统一定义

- `live`：默认指公网测试网 live；
- `public testnet`：当前标准 operating description；
- `tradable`：除非明确说明 venue 和资产，否则只能用于测试网范围；
- `wrapped asset`：默认是测试网表示，不等于主网托管能力；
- `full-loop-tested`：是测试网证据，不是生产安全证明；
- `automatic-loop-ready`：是测试网配置状态，不是 production claim；
- `production`：除非明确命名具体子系统，否则避免泛用；
- `mainnet-candidate`：表示未来门禁，不是当前状态；
- `decentralization`：要保守表述，验证人独立性仍在补；
- `non-custodial`：是目标经营边界，不等于所有外部路由法务已经闭环；
- `settlement`：是技术生命周期 rails，不等于持牌金融清算；
- `compliance-ready`：避免整体化使用，只说已经存在的具体文档或流程。
