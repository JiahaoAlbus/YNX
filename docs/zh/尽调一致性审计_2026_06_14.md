# YNX 尽调一致性审计

状态：active audit note  
最后更新：2026-06-14  
规范语言：中文

## 目的

这份说明记录了挑剔投资人最可能直接交叉核验的内容：README、公开文档和线上
端点之间是否一致。

## 1. Bridge readiness 术语

2026-06-14 实际核验：

- `GET https://rpc.ynxweb4.com/bridge/route-readiness`

观测结果：

- `full_loop_tested = 5/5`
- `automatic_loop_ready = 4/5`

因此对外必须严格区分：

- `automatic_loop_ready` = 配置和 adapter 就绪；
- 当前最强的公开自动化实证仍主要集中在 Sepolia ETH 和 USDC；
- readiness 不能直接被表述成 production safety。

## 2. Validator 语言边界

严格公开测试网 readiness 只能证明：

- 服务在线；
- validator set 活着；
- 签名连续；
- 基本拓扑冗余存在。

它不能证明：

- 强验证人独立性；
- production 级去中心化；
- 多独立 operator 下的治理中立性。

所以投资人材料里应使用“public-testnet validator set live”或“topology redundancy present”，而不是宽泛的去中心化表述。

## 3. 网站 legal 页面缺口

2026-06-14 实际观察：

- `https://www.ynxweb4.com/privacy` 返回网站壳
- `https://www.ynxweb4.com/terms` 返回网站壳

结论：

- 仓库里已经有 legal 文稿；
- 但网站上独立可访问的 legal disclosure 页面仍然是未完成阻塞项。

## 4. Token policy 分离

仓库历史中仍保留较早期的 v0 tokenomics 草案，这些内容不应被表述成当前融资
term sheet。

当前更安全的融资口径是：

- 不默认承诺 founder fee；
- 未来默认 founder fee policy 应保持 `0 bps`，除非单独披露并获批准；
- 融资逻辑应建立在基础设施收入，而不是 token extraction 上。

## 5. 实际结论

YNX 已明显超过“只有概念”的阶段，但投资信心取决于三件事必须持续对齐：

1. 线上端点真相；
2. 尽调材料文字；
3. 官网可见的独立披露页。
