# YNX 尽调一致性审计

状态：active audit note  
最后更新：2026-06-14  
规范语言：中文

## 目的

这份说明记录了挑剔投资人最可能直接交叉核验的内容：README、公开文档和线上
端点之间是否一致。

## 术语权威对照表

| 术语 | 当前最强证据 | 明确不是什么 | 还缺什么门槛 |
|---|---|---|---|
| `live` | 公开 HTTPS 端点和公共测试网服务可响应 | 不是主网、机构级或已审计生产环境 | 持续保持 runtime 证据与文档一致 |
| `public testnet` | `ynx_9102-1`、公开端点、公开 operator 流程 | 不是商业生产网络 | 验证人、审计、法务、灾备门禁 |
| `tradable` | 仅限实际可观察到的公开测试网测试 swap/流动性场景 | 不是“真实外部资产已普遍可交易” | 流动性、风控、法律确认 |
| `wrapped asset` | YNX 上由文档化测试网 route 铸出的测试表示 | 不是持牌托管或主网可赎回资产 | 生产托管、赎回、监控、法务 |
| `full_loop_tested` | 存在 operator 见证的测试网充值、铸币、销毁、释放证据链 | 不是自动化或生产级规模安全证明 | 重复自动化证据与审计 |
| `automatic_loop_ready` | route 配置、watcher、signer 和 adapter 已就绪 | 不是多次公开自动化闭环证明，也不是 production-safe | 真实自动化观察与事故控制 |
| `production` | 当前还不能作为网络级对外结论使用 | 不是当前 public testnet 的同义词 | 审计、可靠性、持久化、法务、治理 |
| `mainnet-candidate` | 截至本审计尚不适用 | 不是任何当前已上线测试网状态 | 全部门禁通过 |
| `decentralization` | 当前只能证明有限拓扑冗余和活着的 validator set | 不是强验证人独立性表述 | 独立 operator 与控制权证据 |
| `non-custodial` | 当前文档中的默认产品/公司边界 | 不是对所有 operated bridge 风险绝对归零的承诺 | 律师确认和运营边界执行 |
| `settlement` | 测试网 job/vault/result/finalize 流程和相关合约 | 不是受监管支付终局或法币结算承诺 | 法律审查与生产控制 |
| `compliance-ready` | 已有文档包和明确待办工作流 | 不是已持牌、已审计、已获律师批准 | 主体、律师、网站发布和控制项闭环 |

## 1. Bridge readiness 术语

2026-06-19 再次核验：

- `GET https://rpc.ynxweb4.com/bridge/route-readiness`

观测结果：

- `deposit_tested = 4/5`
- `release_evidence_observed = 5/5`
- `automatic_loop_ready = 2/5`

因此对外必须严格区分：

- `automatic_loop_ready` = 配置和 adapter 就绪；
- 当前 automatic-ready 的是 BTC testnet BTC 和 TRON Shasta USDT 这两条
  public-testnet release-adapter 路径；
- Sepolia ETH 和 USDC 仍属于 deposit-tested，但还不是 automatic-ready，
  因为线上 bridge service 尚未配置 Sepolia lockbox owner signer；
- readiness 不能直接被表述成 production safety。

## 2. Validator 语言边界

严格公开测试网 readiness 只能证明：

- 服务在线；
- validator set 活着；
- 签名连续；
- 基本拓扑冗余存在。

它不能证明：

- 强验证人独立性；
- production 级去中心化；
- 多独立 operator 下的治理中立性。

所以投资人材料里应使用“public-testnet validator set live”或“topology redundancy present”，而不是宽泛的去中心化表述。

## 3. 网站披露页状态

2026-06-14 实际观察：

- `https://www.ynxweb4.com/privacy` 和 `/terms` 已能被浏览器访问；
- 原始 HTTP 抓取仍会返回 SPA 壳，因此“是否已发布”应以浏览器可达和可读
  内容为准，而不是只看静态 HTML。

结论：

- 官网可见披露路由现在已经存在；
- 剩余 blocker 不再是“有没有这个路由”，而是没有法律实体、没有实体级
  权属/控制人披露、也没有公司级正式政策归属。

## 4. Token policy 分离

仓库历史中仍保留较早期的 v0 tokenomics 草案，这些内容不应被表述成当前融资
term sheet。

当前更安全的融资口径是：

- 不默认承诺 founder fee；
- 未来默认 founder fee policy 应保持 `0 bps`，除非单独披露并获批准；
- 融资逻辑应建立在基础设施收入，而不是 token extraction 上。
- 截至 2026-06-14，公开 indexer overview 仍直接暴露
  `fee_founder_bps = 1000`，因此投资人应要求团队明确解释：为什么当前测试网
  runtime state 与融资口径不完全一致。

## 5. 实际结论

YNX 已明显超过“只有概念”的阶段，但投资信心取决于三件事必须持续对齐：

1. 线上端点真相；
2. 尽调材料文字；
3. 官网可见披露页，以及仓库里对这些披露页状态的描述。
