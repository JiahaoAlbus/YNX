## 5. Public 与 Protected trace 的边界

当前 trace / accountability 是刻意分层的：

- protected trace 可包含：
  - 完整 lot lineage
  - `tainted_amount`
  - 精确 `issuance_id`
  - 精确 `deposit_batch_id`
  - 完整 case evidence / dossier
- public explorer/search 可包含：
  - path preview
  - counterparty directionality
  - root-origin preview
  - provenance anchor 数量摘要
- public explorer/search 应省略：
  - 精确 `issuance_id`
  - 精确 `deposit_batch_id`
  - 可重建级 lot transfer 字段
  - 可重建级 tainted-amount 细节

这不是缺功能，而是有意的安全和隐私边界。
