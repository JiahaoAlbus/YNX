### 2.4 线上 / 本地状态对齐工具

仓库里现在多了一个可重复生成当前状态的命令：

```bash
./scripts/current_full_stack_status_snapshot.sh
```

它会自动产出当前全链 live snapshot。

仓库里现在也多了一个直接做线上 / 本地运行态对齐审计的命令：

```bash
./scripts/verify_live_runtime_alignment.sh
```

它会产出 PASS / WARN / FAIL 级别的对齐报告，直接指出线上部署和当前
本地运行时预期之间还差在哪里。

仓库里现在也多了一个一键 founder/operator 证据包命令：

```bash
./scripts/prepare_full_stack_evidence_pack.sh
```

它会把 latest snapshot、latest alignment、文档校验结果和当前最关键的
状态/合规文档收进一个稳定 handoff 目录和压缩包里。

仓库里现在也多了一个一键 grant / visibility pack 命令：

```bash
./scripts/prepare_grant_visibility_pack.sh
```

它会把 latest truthful status evidence、grant target shortlist、申请模板、
X / Telegram 素材和当前尽调 / 合规边界文档收成一个稳定对外包。

仓库里现在也多了一个最高层的一键 executive closeout pack 命令：

```bash
./scripts/prepare_executive_closeout_pack.sh
```

它会把 snapshot、runtime alignment、founder/operator evidence pack、
grant/visibility pack 再统一收成一个最高层 closeout 目录和压缩包。

仓库里现在也多了一个 closeout packs 验收脚本：

```bash
./scripts/verify_latest_closeout_packs.sh
```

它会校验 latest packs 是否存在、`SHA256SUMS.txt` 是否匹配、
`ARTIFACT_INDEX.json` 是否指向真实文件，以及关键 manifest 链接是否仍然在。
