# YNX Web4 公开测试网赞助简报

## 项目

YNX Web4 Public Testnet 是一个已经上线的公开测试网，核心方向是：

- 受保护的 AI 链上执行
- 跨链测试资产路由
- 基于 Web4 session policy 的授权控制

当前支持边界：

- YNX 仍是项目，不是已经公开设立的法律运营主体；
- 因此这份简报优先面向 grant、赞助和其他项目阶段基础设施支持。

## 问题

AI agent 的执行速度已经超过了人工审批速度。没有 policy 绑定、session 控制和公开可验证的测试网基础设施，团队往往只能在自动化和安全之间二选一。

## 方案

YNX 把下面几件事组合在一起：

- Web4 session authorization
- 受保护的 `trade.execute`
- AI preflight 与 settlement 流程
- 多生态测试网 bridge routing
- 面向操作者和 builder 的 readiness / health 可观测面

`YNX Web4 public testnet is live. We currently have 4/5 bridge routes deposit-tested, 4/5 routes with public release evidence, and 2/5 routes that are fully automatic on the current public-testnet adapter path. ## 为什么现在值得支持

架构不是概念阶段，而是已经在 public testnet 上跑着。接下来最需要的不是证明
“能不能做”，而是把剩余 blocker、网站披露和运营硬化补齐。

资金会直接加速：

- Sepolia release signer 与 BSC lockbox 补齐
- 监控与 incident automation
- validator / infra 冗余
- 外部安全审计准备
- builder 文档与生态接入
- 法务、隐私与公司就绪工作

这份简报不应暗示：

- YNX 已经具备标准股权融资的主体承接条件；
- custody、受监管交易或已审计生产控制已经上线。

Web4 policy enforcement、AI settlement、public infra 和 operator-grade
