# YNX 隐私政策草案

状态：draft  
最后更新：2026-06-13  
规范语言：中文

## 1. 用途

本草案适用于 YNX 的官网、托管 API、开发者服务、支持流程和相关基础设施。

## 2. 可能收集的数据

视服务不同，YNX 可能收集：

- 姓名、邮箱等联系信息；
- 账户和认证元数据；
- API 使用日志；
- 支持和沟通记录；
- 设备、浏览器和网络元数据；
- 用户连接 Web3 工具时的钱包地址或链上标识；
- 分析数据和运维遥测。

## 3. 收集目的

数据可能被用于：

- 运行公开服务；
- 鉴权和 session 安全；
- 监控滥用、欺诈和安全事件；
- 提供支持和服务通知；
- 分析稳定性并改进产品；
- 履行合同或法律义务。

## 4. 数据共享

YNX 可能与以下对象共享数据：

- 云和基础设施供应商；
- 分析、监控和安全服务商；
- 专业顾问和审计方；
- 在法律要求下的主管机关。

默认情况下，YNX 不应以出售个人数据作为商业模式。

## 5. 数据保留

YNX 应只在运营、安全、合规和争议处理所合理必要的期限内保留数据，并在
内部形成正式 retention schedule。

## 6. 安全

YNX 应采用合理的技术和组织措施保护数据，但任何系统都不能保证绝对安全。

## 7. 跨境传输

由于 YNX 可能依赖跨司法辖区的云和服务供应商，数据可能在用户所在地之外被
处理。

## 8. 用户权利

在适用法律赋予访问、更正、删除、撤回同意等权利的情况下，YNX 应提供对应
的联系渠道。

## 9. 测试网声明

即使测试网资产本身没有生产价值，测试网活动仍可能生成日志、支持记录和遥测。

## 10. 联系方式与 DPO

最终发布版本应补充：

- 隐私联系邮箱；
- Data Protection Officer 联系方式；
- 主体名称；
- 经律师确认的司法辖区权利说明。
